Chapter 7: Software Engineering

Term Definition
Software Engineering Seeks principles to guide the development of large, complex software systems.
Integrated Development Environment (IDE) Combine tools for developing software (editors, compilers, debugging tools, and so on) into a single, integrated package.
Association for Computing Machinery (ACM) Purpose: Advance the arts, sciences, and applications of information technology.
Benefits of ACM membership: Access to educational materials as well as social networks to benefit your career.
Student ACM membership: Like normal ACM membership, a student membership is cheaper and focuses on users currently studying computers.
Institute of Electrical and Electronics Engineers (IEEE) Purpose of IEEE: To foster technological innovation and excellence for the benefit of humanity. Focuses on the hardware engineering aspect of computers.
Benefits of IEEE Membership: Access to innovation and cutting-edge technology and networking opportunities.
Student IEEE Membership: Like normal benefits but focused on students and school settings.
Software Life Cycle: Development Phase Requirement Analysis: Specify what services the proposed system will provide, to identify any conditions on those services, and to define how the outside world will interact with the system.
Design: Design involves creating a plan for the construction of the proposed system.
Implementation: Implementation involves the actual writing of programs, creation of data files, and development of databases.
Testing: The traditional development phase of the past, testing was essentially equated with the process of debugging programs and confirming that the final software product was compatible with the software requirements specification.
Modular Implementation Emphasizes separating the functionality of a program into independent, interchangeable modules.
Unified Modeling Language (UML) Provides standard notation for many types of diagrams which can be roughly divided into three main groups: behavior diagrams, interaction diagrams, and structure diagrams.
Use Case A written description of how users will perform tasks on your website.
Use Case Diagram Depicts the proposed system as a large rectangle in which interactions (called use cases) between the system and its users are represented as ovals and users of the system (called actors) are represented as stick figures (even though an actor may not be a person).
Data Relationships One-to-one: A link between the information in two tables, where each record in each table only appears once.
One-to-many: One record in a table can be associated with one or more records in another table.
Many-to-many: When multiple records in a table are associated with multiple records in another table.
Design Pattern A predeveloped model for solving recurring problems in software design.
Software Quality Assurance (SQA) Groups which are charged with overseeing and enforcing the quality control systems adopted by the organization.
Software Testing Glass box testing: The software tester is aware of the interior structure of the software and uses this knowledge when designing the test.
Black box testing: Tests that do not rely on knowledge of the software’s interior composition.
Beta testing: A preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in real-life situations before the final version of the product is solidified and released to the market.
Alpha testing: Real-world testing done at a developer’s site.
Documentation User documentation: Explains features of the software and describe how to use them.
System Documentation: Describes the software’s internal composition so that the software can be maintained later in its life cycle.
Technical Documentation: Describes how a software system should be installed and serviced (such as adjusting operating parameters, installing updates, and reporting problems back to the software’s developer).
Intellectual Property Law Software License: A legal agreement between the owner and user of a software product that grants the user certain permissions to use the product without transferring ownership rights to the intellectual property.
Software Disclaimer: To protect themselves against liability, software developers often include disclaimers in the software licenses that state the limitations of their liability.
