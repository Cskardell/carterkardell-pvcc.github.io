Q: What is a version control system?
    A: A version control system is a practice that manages the changes to source code over time.
Q: What is a git?
    A: A git is a version control system itself that manages the changes to code over time. It gives us the ability to answer "who?""what?"and "why?" questions when looking at code changes.
Q: What is GitHub?
    A: GitHub is a platform that utilizes Git in a cloud based format. Which, makes it much easier for large groups to collaborate on projects.
Q: What is the difference between Git and GitHub?
    A: Git is the software that actually tracks the code changes. GitHub utilizes that software in a cloud based, UI friendly, platform. wh
Q: Who started GitHub and how?
    A: Tom Preston-Werner, Chris Wanstrath, and P.J Hyett were writing code for a computer review site. These three predominately used a software called "Ruby on Rails" for their work. This software had a open source model where "trusted programmers" could add and commit changes to the original Ruby on Rails code to improve it. If another coder wished to impliment their own ideas into Ruby on Rails, they had to petition a trusted coder. That process was not easy, so Tom, Chris, and P.J saw this flaw and began using the relatively unknown Git system to build a platform where coders could easily collaborate. After it's official launch, GitHub took off and recieved increased venture capital funding. Now it is used by more than 14 million people.
Q: What company owns it now?
    A: Microsoft used GitHub so extensively in their own software development, they eventually bought the company.
Q: How much does a GitHub account cost?
    A: A GitHub account is free! However, there are premium packages and features blocked with pay-walls.
Q: What is the "Octocat"?
    The Octocat is the GitHub mascot. It depicts a cat like form with octopus tentacles. It was chosen by the GitHub team supposedly to represent the complex nature of coding.

Git/GitHub Terms

Repository= The place where project files and change logs are stored.
Commit= A save button. Finalizes and logs the changes made to code.
Fork= A copy of a repository sent to another location for open source improvement.
Push= Uploads local repository info. to a remote repository.
Pull Requests= Where discussion is held on a certain change to a repository.
Workflows= A configurated, automated process that will run one or more jobs.
Issues= Track work on GitHub
Raw Button= Lets you copy the raw version of a file without any formatting.
Blame Button= Gives a line by line revision history.
